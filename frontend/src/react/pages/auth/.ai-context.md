# Module: Auth Pages

## Scope
Authentication and authorization flows: sign in, sign up, MFA, SSO callbacks,
password reset/forgot, OAuth2 consent, initial setup.

## Primary Files
- `SigninPage.tsx` — email/password sign in
- `SigninAdminPage.tsx` — admin sign in (first user)
- `SignupPage.tsx` — new user registration
- `SetupPage.tsx` — initial workspace setup wizard
- `PasswordForgotPage.tsx` — password reset request
- `PasswordResetPage.tsx` — password reset form
- `MultiFactorPage.tsx` — MFA verification
- `TwoFactorRequiredPage.tsx` — 2FA enforcement redirect
- `OAuthCallbackPage.tsx` — SSO/OAuth callback handler
- `OAuth2ConsentPage.tsx` — OAuth2 consent screen
- `AuthGuard.tsx` — route guard component (checks auth state)

## Dependencies
- **API Clients**: `authServiceClientConnect`
- **Stores**: `useAuthStore` (login/logout/token management)
- **Routes**: All under `/auth/*` — see `src/router/auth.ts`

## ⚠️ Critical Warnings
- **OAuth callback routes bypass navigation guards** — DO NOT add guards to `/auth/callback` routes
- **Token refresh is handled by interceptors** — DO NOT add manual token refresh logic
- **Cross-tab token coordination** uses Web Locks + BroadcastChannel — see `.ai-context/ERROR_POLICY.md`
- **SetupPage** only renders when no workspace exists — guarded by actuator check

## Common Tasks → File to Edit
| Task | File |
|---|---|
| Add SSO provider | `OAuthCallbackPage.tsx` + IDP config |
| Modify login form | `SigninPage.tsx` |
| Add MFA method | `MultiFactorPage.tsx` |
| Change password rules | `PasswordResetPage.tsx` |
| Modify setup wizard | `SetupPage.tsx` |

## Prohibited
- No `useVueState` calls
- No manual `fetch()` for auth endpoints — use `authServiceClientConnect`
- No storing tokens in localStorage (httpOnly cookies used)
- No bypassing `AuthGuard.tsx` — redirect logic must go through guards
