## 3. Kế hoạch triển khai theo nhóm Database

### Oracle

**Phạm vi:**
- QR Connector
- Payment
- MMS / VMMS
- Ví điện tử
- UMS
- EMB
- SMS
- Topup
- Evoucher

**Nội dung triển khai:**
- Tạo Profile kiểm soát password policy
- Tạo user App mới
- Chuyển đổi ứng dụng sang user mới
- Expire user cũ
- Cấu hình audit login/action

### MongoDB

**Phạm vi:**
- EMB
- Công nợ
- OTT Systems
- Omni
- Marketing

**Nội dung triển khai:**
- Chuẩn hóa account/service account
- Áp dụng password policy
- Rà soát DB Read account
- Audit authentication

### PostgreSQL

**Phạm vi:**
- Omni + Fee
- Billing / Billing HST

**Nội dung triển khai:**
- Tạo role/profile chuẩn
- Kiểm soát password expiration
- Rà soát connection application

### SQLServer

**Phạm vi:**
- BillWeb
- SMS
- Topup

**Nội dung triển khai:**
- Áp dụng CHECK_POLICY / CHECK_EXPIRATION
- Rà soát SQL Login
- Audit login failure/success

## 4. Các hạng mục cần theo dõi

| Nội dung | Trạng thái |
|---|---|
| Tool thông báo password hết hạn | Đang nghiên cứu |
| Audit DB tập trung | Phối hợp ANHT |
| Rà soát hardcode password | Đang thực hiện |
| Chuẩn hóa user service | Đang thực hiện |
| Rollback plan khi đổi user | Cần hoàn thiện |
| Đồng bộ quy trình GSVH / DBA | Đang triển khai |

## 5. Rủi ro triển khai

| Rủi ro | Ảnh hưởng |
|---|---|
| Ứng dụng hardcode DB password | Không kết nối được DB sau đổi password |
| Session tồn đọng user cũ | Không disable được user cũ |
| Downtime dịch vụ | Ảnh hưởng giao dịch |
| Hệ thống legacy | Không hỗ trợ password policy mới |
| Chưa có tool notify | User owner bị hết hạn mà không biết |
