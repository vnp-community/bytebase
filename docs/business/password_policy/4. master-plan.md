## Mục tiêu triển khai

- Chuẩn hóa chính sách mật khẩu Database trên toàn hệ thống
- Đảm bảo tuân thủ yêu cầu audit và security
- Kiểm soát truy cập DB theo nguyên tắc phân quyền tối thiểu
- Tăng khả năng truy vết/audit user database
- Giảm thiểu rủi ro từ tài khoản có mật khẩu yếu hoặc dùng chung
- Chuẩn hóa quy trình cấp phát và quản lý DB User

## Rủi ro và lưu ý triển khai

| Nội dung | Mô tả |
|---|---|
| Downtime dịch vụ | Một số ứng dụng cần restart hoặc downtime khi thay đổi DB user/password |
| Hardcode password | Cần rà soát ứng dụng hardcode DB password |
| Kết nối tồn đọng | Phải kiểm tra session/kết nối từ user cũ sau chuyển đổi |
| Tương thích ứng dụng | Một số ứng dụng cũ có thể không hỗ trợ password policy mới |
| Audit | Cần đảm bảo log đầy đủ thao tác cấp quyền và thay đổi user |
