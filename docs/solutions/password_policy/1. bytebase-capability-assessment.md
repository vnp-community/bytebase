## 1. Tổng quan đánh giá

### 1.1 Bối cảnh Business

VNPAY đang triển khai dự án **chuẩn hóa chính sách mật khẩu và phân quyền User Database** trên toàn hệ thống, bao gồm:

- **90+ dịch vụ Production** trải rộng trên 5 hệ quản trị CSDL (Oracle, PostgreSQL, MongoDB, SQLServer, MySQL)
- **6 nhóm quyền** (Admin, Owner, Service, Operate, Monitor, Viewer) với ma trận phân quyền chi tiết
- **3 phase triển khai** từ 05/2026 đến 09/2026
- Yêu cầu tuân thủ **PCI-DSS** và **ISO27001**

### 1.2 Tổng kết khả năng hỗ trợ

| Mức độ hỗ trợ | Số yêu cầu | Tỷ lệ |
|---|---|---|
| ✅ **Hỗ trợ trực tiếp** (Native) | 12 | 40% |
| 🟡 **Hỗ trợ một phần** (Partial) | 10 | 33% |
| 🔴 **Không hỗ trợ** (cần bổ sung open source) | 8 | 27% |

## 2. Mapping chi tiết: Yêu cầu Business ↔ Bytebase Feature

### 2.1 Chính sách mật khẩu (Business Doc #1)

| Yêu cầu Business | Bytebase Feature | Mức độ | Ghi chú |
|---|---|---|---|
| Độ dài mật khẩu tối thiểu 12 ký tự | SEC-13: Password Restrictions (Enterprise) | 🟡 Partial | Chỉ áp dụng cho **Bytebase user**, KHÔNG áp dụng cho database user trên target DB |
| Độ phức tạp mật khẩu (chữ hoa, chữ thường, số, đặc biệt) | SEC-13: Password Restrictions (Enterprise) | 🟡 Partial | Tương tự — chỉ cho Bytebase login |
| Đổi mật khẩu khi đăng nhập lần đầu | — | 🔴 None | Bytebase không quản lý database user password lifecycle |
| Hết hạn mật khẩu 90 ngày | — | 🔴 None | Cần native DB profile/policy hoặc tool bên ngoài |
| Khóa tài khoản sau X lần đăng nhập sai | — | 🔴 None | Cần cấu hình ở DB engine level |

> **Kết luận**: Bytebase **không quản lý password policy cho database user** trên target databases. Chức năng SEC-13 chỉ áp dụng cho chính Bytebase platform. Cần sử dụng **native DB profile/policy** (Oracle Profile, PostgreSQL pgaudit, MySQL validate_password plugin) hoặc tool bên ngoài.

### 2.2 Ma trận phân quyền Database (Business Doc #2)

| Yêu cầu Business | Bytebase Feature | Mức độ | Ghi chú |
|---|---|---|---|
| 6 nhóm quyền (Admin, Owner, Service, Operate, Monitor, Viewer) | SEC-01: IAM + SEC-14: Custom Roles (Enterprise) | 🟡 Partial | Bytebase RBAC quản lý quyền **truy cập Bytebase**, không quản lý trực tiếp DB-level GRANT/REVOKE |
| Nguyên tắc Least Privilege | SEC-01 + SEC-09: Approval Workflow | ✅ Full | Kiểm soát ai được thực thi SQL gì thông qua approval |
| Kiểm soát DDL/DCL phải phê duyệt | DCM-01: Database Change Pipeline + SEC-09 | ✅ Full | Tất cả DDL/DCL đi qua pipeline Issue → Plan → Approve → Rollout |
| Log/audit trail thao tác cấp quyền | SEC-10: Audit Log (Enterprise) | ✅ Full | Ghi log toàn bộ API calls, bao gồm thay đổi permission |
| Rà soát quyền định kỳ | — | 🔴 None | Cần workflow bên ngoài hoặc script định kỳ |

> **Kết luận**: Bytebase xuất sắc trong việc **kiểm soát ai được phép thực thi SQL** (approval workflow), nhưng việc **GRANT/REVOKE permission trên database level** phải thông qua SQL statements trong Bytebase pipeline, không phải tính năng native.

## 5. Kết luận

### Điểm mạnh của Bytebase cho dự án VNPAY

1. **Centralized Change Management** — Tất cả thay đổi DB (CREATE USER, ALTER PROFILE, GRANT) đều đi qua pipeline kiểm soát
2. **Multi-DB Engine Support** — Hỗ trợ native cả 5 loại DB mà VNPAY sử dụng
3. **Approval Workflow** — Phù hợp với quy trình phê duyệt multi-level (Trưởng phòng → DBA → PO)
4. **Audit Trail** — Ghi log đầy đủ mọi thao tác, đáp ứng PCI-DSS/ISO27001
5. **Batch Changes** — Triển khai đồng loạt cho từng phase với nhiều dịch vụ

### Hạn chế cần bổ sung

1. **Không quản lý native DB password policy** — Cần kết hợp với DB engine profiles
2. **Không monitoring sau triển khai** — Cần stack Grafana/Prometheus
3. **Không thông báo password expiry** — Cần tool notification bên ngoài
4. **Không có periodic access review** — Cần script/workflow bên ngoài
