## 1. Tổng quan

### 1.1 Mô tả
Agent tích hợp Steampipe để thực hiện policy-as-code access auditing và đồng bộ kết quả vào Bytebase. Agent chủ động:
- Chạy Steampipe compliance checks định kỳ
- Import audit findings vào Bytebase Access Review (CR-INS-004)
- Maintain Steampipe mods/queries synchronized với Bytebase permission matrix
- Generate evidence reports cho PCI-DSS/ISO27001 compliance

### 1.2 Bối cảnh
Steampipe cung cấp SQL-based policy-as-code framework mạnh hơn custom scripts. Agent này orchestrates Steampipe execution và bridges kết quả vào Bytebase workflow.

### 1.3 Mục tiêu
- Automated policy-as-code access auditing
- Steampipe mods tự động generated từ Bytebase permission matrix
- Audit findings → Bytebase remediation workflow
- Cross-platform evidence collection

## 2. Yêu cầu chức năng

### FR-001: Steampipe Mod Generator
- Read Bytebase permission matrix (CR-INS-004) → generate Steampipe mods
- Generated checks:
  - `control.no_superuser_except_admin` — Chỉ Admin group có superuser
  - `control.privilege_matches_matrix` — User privileges match matrix
  - `control.no_orphaned_users` — All DB users map to Bytebase users
  - `control.password_policy_applied` — All users have proper profile
  - `control.no_cross_schema_access` — No unauthorized cross-schema grants
- Auto-update mods khi matrix changes trong Bytebase

### FR-002: Steampipe Connection Manager
- Auto-configure Steampipe connections từ Bytebase instance registry
- Connection config generated per engine:
  - `connection "postgres_prod" { plugin = "postgres" host = "..." }`
- Credentials from Bytebase's External Secret Manager
- Connection health verification before audit runs

### FR-003: Scheduled Audit Execution
- Cron-based execution (weekly/monthly/quarterly configurable)
- Steampipe `check` command execution
- Output: JSON/CSV + Steampipe dashboard snapshot
- Results persistence: store audit history for trend analysis

### FR-004: Findings Import & Remediation Bridge
- Parse Steampipe check results → import vào Bytebase
- Findings classification:
  - `ALARM` → Bytebase violation (auto-create remediation Issue)
  - `OK` → Compliant (update compliance score)
  - `SKIP` → Not applicable
  - `ERROR` → Agent diagnostic issue
- Remediation Issue: pre-populated REVOKE/ALTER SQL

### FR-005: Evidence Export
- Package audit results as compliance evidence:
  - Timestamped reports
  - Signed with Bytebase audit chain
  - Format: PDF + machine-readable JSON
- Attach to Bytebase compliance report (CR-INS-008)

### 3.1 Generated Steampipe Mod Example

```hcl
# auto-generated by Bytebase Steampipe Agent
mod "vnpay_db_access_audit" {
  title = "VNPAY Database Access Audit"
}

control "no_superuser_except_admin" {
  title = "Only Admin group users have superuser privileges"
  sql = <<-EOQ
    SELECT
      usename as resource,
      CASE WHEN usename IN ('bytebase_admin', 'postgres')
        THEN 'ok' ELSE 'alarm'
      END as status,
      usename || ' has superuser privilege' as reason
    FROM pg_user
    WHERE usesuper = true
  EOQ
}

control "privilege_matches_matrix" {
  title = "User privileges match VNPAY permission matrix"
  sql = <<-EOQ
    SELECT
      grantee as resource,
      CASE WHEN privilege_type IN ('INSERT','UPDATE','DELETE')
        AND grantee NOT IN (${join(", ", [for u in var.allowed_write_users : "'${u}'"])})
        THEN 'alarm' ELSE 'ok'
      END as status,
      grantee || ' has ' || privilege_type || ' on ' || table_schema as reason
    FROM information_schema.role_table_grants
    WHERE grantee NOT IN ('postgres', 'bytebase_admin')
  EOQ
}
```
