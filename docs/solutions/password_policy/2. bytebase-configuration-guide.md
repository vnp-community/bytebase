## 1. Triển khai Bytebase

### 1.1 Docker Deployment (Khuyến nghị cho Production)

```bash
# Tạo thư mục data
mkdir -p /opt/bytebase/data

# Chạy Bytebase với external PostgreSQL (HA mode)
docker run -d \
  --name bytebase \
  --restart always \
  --publish 443:8080 \
  --volume /opt/bytebase/data:/var/opt/bytebase \
  -e PG_URL="postgresql://bytebase:PASSWORD@pg-host:5432/bytebase?sslmode=require" \
  bytebase/bytebase:latest
```

### 1.2 Kích hoạt Enterprise License

```
Settings → Subscription → Nhập Enterprise License Key
```

> **Bắt buộc Enterprise** cho: Approval Workflow, Full Audit Log, Custom Roles, Password Restrictions, SSO.

## 2. Cấu hình Environment Hierarchy

### 2.1 Tạo Environments

Tạo 3 environment tương ứng với quy trình triển khai VNPAY:

```
Settings → Environments → Add Environment
```

| Environment | Order | Tier | Rollout Policy | Mô tả |
|---|---|---|---|---|
| `Test` | 1 | Development | Automatic | Môi trường kiểm thử |
| `Staging` | 2 | Staging | Manual (DBA approve) | Môi trường pre-prod |
| `Production` | 3 | Production | Manual (Multi-level approve) | Môi trường production |

### 2.2 Cấu hình Environment Tier (Enterprise)

```
Settings → Environments → Production → Tier = Production
```

Khi đặt tier Production, Bytebase sẽ:
- Yêu cầu approval cho mọi thay đổi
- Tự động chạy SQL Review trước deploy
- Ghi audit log chi tiết hơn

## 6. Cấu hình Approval Workflow

### 6.1 Risk Assessment Rules (Enterprise)

```
Settings → Security & Policy → Risk Assessment
```

| Rule | Condition | Risk Level |
|---|---|---|
| DDL on Production | `environment.tier == "PRODUCTION" && statement.type in ["DDL"]` | HIGH |
| CREATE/ALTER USER | `statement contains "CREATE USER" OR "ALTER USER"` | HIGH |
| GRANT/REVOKE | `statement contains "GRANT" OR "REVOKE"` | HIGH |
| DML on Production | `environment.tier == "PRODUCTION" && statement.type in ["DML"]` | MODERATE |
| DDL on Staging | `environment.tier == "STAGING" && statement.type in ["DDL"]` | MODERATE |

### 6.2 Approval Policy

```
Settings → Security & Policy → Custom Approval
```

**HIGH Risk** (CREATE USER, GRANT, Production DDL):

```yaml
Approval Flow: "Production DB User Management"
Steps:
  1. Trưởng phòng đầu mối (VNPAY_DB_Owner role)
  2. DBA Admin (VNPAY_DBA_Admin role)
  3. PO / ANHT (cho yêu cầu từ đối tác ngoài)
```

**MODERATE Risk** (DML Production, DDL Staging):

```yaml
Approval Flow: "Standard Change"
Steps:
  1. DBA Admin (VNPAY_DBA_Admin role)
```

**LOW Risk** (Test environment):

```yaml
Approval Flow: "Auto Approve"
Auto-approve: true
```

## 7. Cấu hình SQL Review Policy

### 7.1 Tạo SQL Review Config cho mỗi Engine

```
Settings → SQL Review → Add Policy
```

#### Oracle SQL Review Policy

```
Policy Name: VNPAY-Oracle-Standard
Target: Oracle instances
Rules (enable):
  - naming.table (table naming convention)
  - naming.column (column naming convention)
  - statement.disallow-cross-db-queries
  - statement.require-where (DML must have WHERE)
  - column.no-null (columns should have NOT NULL)
  - index.no-duplicate (no duplicate indexes)
  # Custom rules cho password policy
  - statement.custom-rule: "CREATE USER phải sử dụng PROFILE"
```

#### PostgreSQL SQL Review Policy

```
Policy Name: VNPAY-PostgreSQL-Standard
Target: PostgreSQL instances
Rules (enable):
  - naming.table
  - naming.column
  - statement.require-where
  - column.required-column (require audit columns: created_at, updated_at)
  - index.primary-key-type (prefer UUID or BIGINT)
```

### 7.2 Template SQL cho tạo User (dùng trong pipeline)

Tạo **SQL Templates** (Sheets) chuẩn cho DBA sử dụng:

#### Oracle — Tạo User với Profile

```sql
-- Sheet: oracle-create-user-template
-- Tạo User Owner với User_Profile
CREATE USER {{username}} IDENTIFIED BY "{{temp_password}}"
  DEFAULT TABLESPACE users
  TEMPORARY TABLESPACE temp
  PROFILE USER_PROFILE;

GRANT CONNECT, RESOURCE TO {{username}};
ALTER USER {{username}} PASSWORD EXPIRE;

-- Tạo User Service với Service_Profile
CREATE USER {{svc_username}} IDENTIFIED BY "{{temp_password}}"
  DEFAULT TABLESPACE users
  TEMPORARY TABLESPACE temp
  PROFILE SERVICE_PROFILE;

GRANT CONNECT TO {{svc_username}};
```

#### PostgreSQL — Tạo User với Policy

```sql
-- Sheet: pg-create-user-template
-- Tạo User Owner
CREATE ROLE {{username}} WITH
  LOGIN
  PASSWORD '{{temp_password}}'
  VALID UNTIL '{{expiry_date}}'
  CONNECTION LIMIT 10;

-- Tạo User Service (không expire)
CREATE ROLE {{svc_username}} WITH
  LOGIN
  PASSWORD '{{temp_password}}'
  CONNECTION LIMIT 50;

-- Grant schema access
GRANT USAGE ON SCHEMA {{schema}} TO {{username}};
GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA {{schema}} TO {{username}};
```

## 8. Cấu hình Audit Logging

### 8.1 Enable Full Audit Log (Enterprise)

```
Settings → Security & Policy → Audit Log
- Level: Full
- Retention: 365 days (đáp ứng PCI-DSS yêu cầu lưu 1 năm)
```

### 8.2 Webhook Integration cho Notification

```
Settings → Integration → Webhook
```

| Webhook | Target | Events |
|---|---|---|
| Slack DBA Channel | `#dba-alerts` | Issue created, Approval needed, Rollout completed |
| Jira Integration | Jira webhook URL | Issue created (tạo Jira ticket tương ứng) |
| Email DBA | dba-team@vnpay.vn | Approval required, Rollout failed |

### 8.3 Export Audit Log cho Compliance

```
Audit Log → Filter → Export
- Filter by: action = "bb.sql.execute", time range
- Format: CSV / JSON
- Schedule: Monthly export cho compliance team
```

## 9. Workflow chuẩn: Triển khai Password Policy qua Bytebase

### 9.1 Luồng tạo Profile/Rule trên DB (Bước II.1 Master Plan)

```
1. DBA đăng nhập Bytebase
2. Project → Select "DB-Password-Policy"
3. Create Issue:
   - Title: "[Phase1] Tạo Password Profile trên Oracle - QR Connector"
   - Database Group: Phase1-Oracle
   - SQL Type: DDL
   
4. Nhập SQL (sử dụng template):
   -- Tạo USER_PROFILE (90 ngày expired)
   CREATE PROFILE USER_PROFILE LIMIT
     PASSWORD_LIFE_TIME 90
     PASSWORD_REUSE_MAX 5
     PASSWORD_REUSE_TIME 365
     FAILED_LOGIN_ATTEMPTS 5
     PASSWORD_LOCK_TIME 1/24
     PASSWORD_VERIFY_FUNCTION ora12c_verify_function;
   
   -- Tạo SERVICE_PROFILE (không expired)
   CREATE PROFILE SERVICE_PROFILE LIMIT
     PASSWORD_LIFE_TIME UNLIMITED
     PASSWORD_REUSE_MAX UNLIMITED
     FAILED_LOGIN_ATTEMPTS 10
     PASSWORD_LOCK_TIME 1/24
     PASSWORD_VERIFY_FUNCTION ora12c_verify_function;

5. Submit → SQL Review tự động chạy
6. Trưởng phòng review → Approve
7. DBA Admin review → Approve
8. Rollout tự động thực thi trên tất cả Oracle instances trong group
9. Changelog được cập nhật
10. Audit log ghi nhận toàn bộ
```

### 9.2 Luồng tạo User mới (Bước II.3 Master Plan)

```
1. DBA tạo Issue:
   - Title: "[Phase1] Tạo User App mới - QR_CONNECTOR_APP1"
   - Database: QR Connector Oracle (Production)
   
2. SQL:
   CREATE USER QR_CONNECTOR_APP1 IDENTIFIED BY "TempP@ss2026!"
     DEFAULT TABLESPACE USERS
     PROFILE SERVICE_PROFILE;
   
   GRANT CONNECT TO QR_CONNECTOR_APP1;
   GRANT SELECT, INSERT, UPDATE, DELETE ON QR_CONNECTOR.* TO QR_CONNECTOR_APP1;
   ALTER USER QR_CONNECTOR_APP1 PASSWORD EXPIRE;

3. Approval Flow: HIGH Risk → 2 cấp phê duyệt
4. Sau approve → Rollout thực thi
5. DBA bàn giao credentials qua Vault
```

### 9.3 Luồng Expire User cũ (Bước 3.4 Deploy Process)

```
1. DBA tạo Issue:
   - Title: "[Phase1] Expire User App cũ - QR_CONNECTOR_APP"
   
2. SQL:
   ALTER USER QR_CONNECTOR_APP ACCOUNT LOCK;
   -- Theo dõi 7 ngày, sau đó:
   -- ALTER USER QR_CONNECTOR_APP PASSWORD EXPIRE;

3. Approval → DBA Admin
4. Rollout → Monitor kết nối 7 ngày
5. Tạo Issue tiếp theo để hoàn tất lock
```
