# Module: Project Pages

## Scope
Project-level features: databases, plans, issues, releases, settings, members,
data masking, schema sync, webhooks, access grants, audit log, data export.

## Primary Files
- `ProjectLandingPage.tsx` ‚Äî project dashboard
- `ProjectDatabasesPage.tsx` ‚Äî database list for project
- `ProjectDatabaseDetailPage.tsx` ‚Äî single database detail
- `ProjectPlanDashboardPage.tsx` ‚Äî plan list
- `ProjectPlanDetailPage.tsx` ‚Äî plan detail + spec editing
- `ProjectIssueDashboardPage.tsx` ‚Äî issue list
- `ProjectIssueDetailPage.tsx` ‚Äî issue detail + approval flow
- `ProjectReleaseDashboardPage.tsx` ‚Äî release list
- `ProjectReleaseDetailPage.tsx` ‚Äî release detail
- `ProjectSettingsPage.tsx` ‚Äî project settings
- `ProjectGitOpsPage.tsx` ‚Äî VCS integration
- `ProjectSyncSchemaPage.tsx` ‚Äî schema sync wizard
- `ProjectWebhooksPage.tsx` ‚Äî webhook list
- `ProjectWebhookCreatePage.tsx` / `ProjectWebhookDetailPage.tsx` ‚Äî webhook CRUD
- `ProjectWebhookForm.tsx` ‚Äî shared webhook form
- `ProjectAccessGrantsPage.tsx` ‚Äî access grant management
- `ProjectMaskingExemptionPage.tsx` ‚Äî masking exemption list
- `ProjectMaskingExemptionCreatePage.tsx` ‚Äî create masking exemption
- `ProjectAuditLogPage.tsx` ‚Äî project-scoped audit log
- `ProjectDataExportPage.tsx` ‚Äî data export center
- `ProjectDatabaseGroupsPage.tsx` ‚Äî database group list
- `ProjectDatabaseGroupCreatePage.tsx` / `ProjectDatabaseGroupDetailPage.tsx` ‚Äî group CRUD
- `DatabaseChangelogDetailPage.tsx` ‚Äî changelog detail
- `DatabaseRevisionDetailPage.tsx` ‚Äî revision detail

## Subdirectories
- `database-detail/` ‚Äî database detail sub-pages (schema, data, changelog)
- `plan-detail/` ‚Äî plan detail components
- `issue-detail/` ‚Äî issue detail components (approval, task status, timeline)
- `export-center/` ‚Äî data export components

## Dependencies
- **API Clients**: `projectServiceClientConnect`, `databaseServiceClientConnect`, `planServiceClientConnect`, `issueServiceClientConnect`, `rolloutServiceClientConnect`, `releaseServiceClientConnect`, `accessGrantServiceClientConnect`, `orgPolicyServiceClientConnect`
- **Stores**: `useAuthStore` (project-level permissions), `useSubscriptionV1Store` (feature gating)
- **Permissions**: project-level `bb.projects.*`, `bb.databases.*`, `bb.issues.*`, `bb.plans.*`

## Common Tasks ‚Üí File to Edit
| Task | File |
|---|---|
| Add issue filter | `ProjectIssueDashboardPage.tsx` |
| Modify plan spec | `plan-detail/` subdirectory |
| Add database column | `ProjectDatabasesPage.tsx` |
| Edit approval flow | `issue-detail/` subdirectory |
| Add release field | `ProjectReleaseDetailPage.tsx` |
| Schema sync config | `ProjectSyncSchemaPage.tsx` |

## Prohibited
- No `useVueState` calls ‚Äî project pages are fully React
- No raw `z-index` ‚Äî use overlay system
- All API calls via ConnectRPC clients
- `updateMask` required for all update operations
