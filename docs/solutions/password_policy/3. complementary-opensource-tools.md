## 2. Chi tiết từng Tool

### 2.1 Percona Monitoring and Management (PMM)

**Giải quyết**: G3 (Session monitoring), G5 (Performance monitoring)

| Thuộc tính | Chi tiết |
|---|---|
| License | AGPL v3 (Open Source) |
| Repository | github.com/percona/pmm |
| Hỗ trợ DB | MySQL, PostgreSQL, MongoDB, ProxySQL |
| Deploy | Docker / K8s |

**Tính năng phù hợp VNPAY**:
- **Query Analytics** — theo dõi query performance sau chuyển đổi user
- **Active session monitoring** — theo dõi kết nối user cũ/mới real-time
- **Security Threat Tool** — phát hiện user có password yếu trên MySQL/PG
- **Dashboard** — Grafana-based, custom được theo nhu cầu

**Triển khai**:
```bash
docker run -d --name pmm-server \
  -p 443:443 \
  --volume pmm-data:/srv \
  percona/pmm-server:2

# Cài PMM client trên mỗi DB server
pmm-admin add postgresql --username=pmm --password=xxx --host=10.20.4.254
pmm-admin add mongodb --username=pmm --password=xxx --host=10.20.4.x
```

**Giám sát session user cũ** (PostgreSQL):
```sql
-- Custom query cho PMM dashboard
SELECT usename, client_addr, state, query_start, now() - query_start as duration
FROM pg_stat_activity
WHERE usename IN ('old_app_user', 'old_owner_user')
ORDER BY query_start;
```

### 2.2 Grafana + Prometheus + DB Exporters

**Giải quyết**: G5 (Performance), G8 (Compliance dashboard)

| Component | Repository | Mục đích |
|---|---|---|
| Grafana | github.com/grafana/grafana | Dashboard & visualization |
| Prometheus | github.com/prometheus/prometheus | Metrics collection |
| oracle_exporter | github.com/iamseth/oracledb_exporter | Oracle metrics |
| postgres_exporter | github.com/prometheus-community/postgres_exporter | PostgreSQL metrics |
| mongodb_exporter | github.com/percona/mongodb_exporter | MongoDB metrics |
| mysqld_exporter | github.com/prometheus/mysqld_exporter | MySQL metrics |
| mssql_exporter | github.com/burningalchemist/sql_exporter | SQL Server metrics |

**Custom metrics cho VNPAY**:

```yaml
# prometheus/custom_queries_oracle.yml
- query: |
    SELECT username, account_status, expiry_date,
      ROUND(expiry_date - SYSDATE) as days_until_expiry
    FROM dba_users
    WHERE profile IN ('USER_PROFILE','SERVICE_PROFILE')
  metrics:
    - username: {usage: "LABEL"}
    - account_status: {usage: "LABEL"}
    - days_until_expiry: {usage: "GAUGE", description: "Days until password expiry"}
```

```yaml
# prometheus/custom_queries_pg.yml
- query: |
    SELECT usename, valuntil,
      EXTRACT(DAY FROM valuntil - now()) as days_until_expiry
    FROM pg_user
    WHERE valuntil IS NOT NULL
  metrics:
    - usename: {usage: "LABEL"}
    - days_until_expiry: {usage: "GAUGE"}
```

**Grafana Dashboard panels**:
- Password Expiry Countdown (all users, all engines)
- Active Sessions per User (old vs new)
- Failed Login Attempts (security)
- Compliance Status (phase completion %)

### 2.3 Password Expiry Notification (Custom Script)

**Giải quyết**: G2 (Password expiration notification)

Không có tool open source dedicated cho cross-DB password notification. Đề xuất **custom script** kết hợp Grafana Alerting:

**Script kiểm tra Oracle password expiry**:
```bash
#!/bin/bash
# check_oracle_password_expiry.sh
# Chạy daily qua cron

THRESHOLD_DAYS=14

sqlplus -S bytebase_admin/xxx@ORCL <<EOF | while read user days; do
  if [ "$days" -le "$THRESHOLD_DAYS" ]; then
    curl -X POST "https://hooks.slack.com/services/xxx" \
      -H 'Content-type: application/json' \
      -d "{\"text\":\"⚠️ DB User '$user' password expires in $days days\"}"
  fi
done
SELECT username, ROUND(expiry_date - SYSDATE) days_left
FROM dba_users
WHERE expiry_date IS NOT NULL
  AND expiry_date - SYSDATE <= $THRESHOLD_DAYS
  AND account_status = 'OPEN';
EOF
```

**Grafana Alert Rule** (thay thế script):
```yaml
# Grafana Alert trên metric days_until_expiry
Alert: DB Password Expiry Warning
Condition: days_until_expiry < 14
Notification: Slack #dba-alerts + Email dba-team@vnpay.vn
Labels:
  severity: warning
  team: dba
```

### 2.4 GitLeaks — Hardcode Password Scanner

**Giải quyết**: G7 (Hardcode password scanning)

| Thuộc tính | Chi tiết |
|---|---|
| License | MIT |
| Repository | github.com/gitleaks/gitleaks |

**Triển khai**:
```bash
# Scan toàn bộ repo ứng dụng
gitleaks detect --source=/path/to/app-repo --report-path=gitleaks-report.json

# Tích hợp CI/CD (GitLab CI)
gitleaks:
  stage: security
  image: zricethezav/gitleaks:latest
  script:
    - gitleaks detect --source=. --report-format=sarif --report-path=gitleaks.sarif
  artifacts:
    reports:
      sast: gitleaks.sarif
```

**Custom rules cho DB passwords**:
```toml
# .gitleaks.toml
[[rules]]
id = "db-password-oracle"
description = "Oracle DB password in config"
regex = '''(?i)(oracle|orcl|jdbc:oracle).*password\s*[=:]\s*['"]?[^\s'"]+'''
tags = ["database", "oracle"]

[[rules]]
id = "db-password-postgres"
description = "PostgreSQL connection string with password"
regex = '''postgresql://[^:]+:([^@]+)@'''
tags = ["database", "postgresql"]
```

### 2.5 Steampipe — Periodic Access Review

**Giải quyết**: G4 (Periodic access review)

| Thuộc tính | Chi tiết |
|---|---|
| License | AGPL v3 |
| Repository | github.com/turbot/steampipe |
| Plugins | PostgreSQL, MySQL, SQLite |

**Query rà soát quyền** (chạy định kỳ):
```sql
-- Steampipe query: Liệt kê tất cả user có quyền DBA/Admin
SELECT usename, usesuper, usecreatedb, valuntil
FROM pg_user
WHERE usesuper = true
ORDER BY usename;

-- Check user nào có quyền vượt quá matrix
SELECT grantee, privilege_type, table_schema
FROM information_schema.role_table_grants
WHERE grantee NOT IN ('postgres', 'bytebase_admin')
  AND privilege_type IN ('INSERT', 'UPDATE', 'DELETE')
  AND table_schema NOT IN ('pg_catalog', 'information_schema');
```
